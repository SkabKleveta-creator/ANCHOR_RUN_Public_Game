# Anchor Run — Acceptance Checklist

Use this checklist before marking a build as accepted.

## Boot

- [ ] Public URL loads without a 404.
- [ ] Title screen appears.
- [ ] Begin Curated Route button starts the game.
- [ ] No console-breaking error freezes the game.

## Desktop controls

- [ ] A / Left Arrow moves left.
- [ ] D / Right Arrow moves right.
- [ ] Space grapples to anchors.
- [ ] Shift boosts.
- [ ] J / Enter interacts with recharge stations and terminals.
- [ ] Esc pauses and resumes.

## Mobile controls

- [ ] L and R move Eli.
- [ ] GRAP grapples.
- [ ] BOOST boosts.
- [ ] ACT interacts.
- [ ] Controls do not overlap the canvas.
- [ ] Controls remain reachable in portrait view.
- [ ] Browser chrome does not hide the lower controls.

## Gameplay

- [ ] Sector 1 can be cleared.
- [ ] At least three sectors can be cleared in sequence.
- [ ] Final Access Core can be reached.
- [ ] Three release switches are required in the Final Access Core.
- [ ] Full run can show RUN RESTORED.

## Guardrails

- [ ] No jump exists.
- [ ] Ladders remain environmental only.
- [ ] No combat appears.
- [ ] No inventory appears.
- [ ] No skill tree appears.
- [ ] No dialogue system appears.
- [ ] No bosses appear.

## Acceptance note

Record device, browser, link, completion result, time, and strain events before calling the build accepted.
